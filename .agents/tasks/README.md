# 任务索引

## Task 创建规则

1. 由 LLM 根据任务复杂度、规模和记录价值自主判断是否创建 task。
2. 建议创建：跨 frontend/backend、涉及多模块、周期长、包含可复用架构决策的任务。
3. 无需创建：简单明了的小改动、一次性修复、没有长期记录价值的任务，直接完成即可。

## 状态流转

1. active：正在执行。
2. completed：实现和验证完成，等待归档。
3. archived：已沉淀结论并移入 archive/。
4. deleted：可选，仅在确认无保留价值时执行。

## 归档规则

1. 完成或长时间未更新的 task，移动到 archive/<task-slug>/。
2. 归档时保留 decisions 和关键结论，删除冗余过程记录。
3. 若 task 仍有后续工作，复用原 task 或新建续作 task，避免同一主题碎片化。

## 压缩规则

1. 合并重复背景、过期计划和重复进度。
2. context.md 只保留长期有效信息，不记录一次性探索日志。
3. decisions.md 只记录最终结论、背景和原因，不堆积中间试错细节。
