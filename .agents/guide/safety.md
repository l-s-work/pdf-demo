# 安全约束

## 禁止直接执行的操作

1. 禁止无确认删除 backend/storage、SQLite 数据库、上传文件、OCR 缓存或 .agents/tasks/archive 中的历史记录。
2. 禁止无确认修改稳定接口语义：GET /api/highlight-hits、GET /api/pdf/{id}/meta、GET /api/pdf/{id}/file。
3. 禁止破坏 per-hit 单矩形语义、服务端原始坐标语义或 Range 支持。
4. 禁止把 OSS 密钥、环境变量密文、测试凭据写入仓库文件。
5. 除非任务明确要求，禁止直接编辑 frontend/dist、frontend/public/pdfjs 下的第三方产物和缓存文件。
6. 禁止使用破坏性 git 命令覆盖用户未授权的本地改动。

## 需要确认的高风险操作

1. 新增或替换核心依赖。
2. 调整接口契约、数据库结构、存储路径或 OSS 行为。
3. 批量清理缓存、历史任务、PDF 文件或命中数据。
4. 引入新的国际化体系、鉴权体系或后台任务调度方式。

## 数据与协作约束

1. 只读取完成任务所需的最小数据范围，不随意遍历或导出存储目录内容。
2. 如果发现工作区已有用户改动且与当前任务冲突，先停下并确认处理方式。
3. 所有验证优先采用局部命令，避免对大型 OCR 资源和存储目录做高成本操作。
