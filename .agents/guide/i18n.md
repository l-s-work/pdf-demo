# 国际化规则

## 当前判断

1. 当前仓库未发现 i18next、react-intl、vue-i18n 等国际化依赖。
2. 当前仓库未发现 locale/locales 目录，也未发现稳定的翻译函数调用模式。
3. 因此本项目目前没有正式 i18n 基础设施，本文件默认按需读取。

## 无 i18n 基础时的处理

1. 非文案任务可以跳过本文件。
2. 涉及用户可见文案、错误提示、按钮文案、接口消息或翻译需求时，必须先读取本文件。
3. 在未确认要引入 i18n 体系前，不要半途混入新的翻译框架或临时 key 方案。
4. 维持现有中文表达风格，避免同一页面或接口消息中中英混杂。

## 若未来引入 i18n，必须遵守

1. 禁止硬编码新的用户可见文案到组件和页面逻辑中。
2. 禁止使用 fallback 写法，例如 t('key') || '默认文案'。
3. key 必须先确认存在后再使用，优先复用已有翻译而不是随意新增近义 key。
4. 是否引入 i18n 体系属于高影响决策，需先确认并记录到 decisions.md。
