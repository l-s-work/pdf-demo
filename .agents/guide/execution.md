# 执行协议

## 基本原则

1. Exploration First：开始修改前，先读取目标目录的 AGENTS、相关 README、相邻实现和直接调用链，只收集足够支撑当前任务的上下文。
2. Plan First：复杂任务、跨模块任务、涉及契约或依赖决策的任务，先写计划再执行；简单且一次性的小改动可直接完成，不强制创建 task 文件。
3. Minimal Diff：优先做最小可验证改动，不在当前任务范围外重构 backend、frontend、docs 或存储结构。
4. Scope First：每次开始实现前声明作用域是 backend、frontend 还是 backend+frontend，并按路径遵守对应子目录 AGENTS。

## 执行流程

1. 先确认入口锚点：文件、接口、页面、命令或失败现象。
2. 只做一轮局部探索，形成可验证假设后立即修改，不做大范围漫游式阅读。
3. 首次实质改动后，立刻执行最便宜的局部验证：相关测试、构建、类型检查、接口检查或结构检查。
4. 验证失败时先修当前切片并复验；只有假设被证伪时才跳到相邻控制点继续定位。
5. 完成后补充必要文档或记忆，但只记录“代码本身难以表达的信息”。

## Task 创建规则

1. 复杂、跨模块、持续时间长、包含值得复用的架构决策时，创建 .agents/tasks/<task-slug>.md。
2. 简单明确、一次性交付、没有长期记录价值的小任务，直接完成即可，不额外建 task。
3. task 中保留计划、探索、决策和进度；完成后按规则归档到 archive/。
